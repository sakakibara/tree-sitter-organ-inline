# Underline emphasis (`_…_`).
# Shares scanner code with subscript via early-return paths, but the
# final `_` <other> path now applies the same post-char rule used by
# bold/italic/strike.

+ _under_
+ a _under_ b
+ (_under_)

# Subscript context: `H_2` and `H_2O` should NOT open underline (no
# closing `_`, alpha-only run after `_` is subscript).
- H_2O
- x_1
# No close.
- _under
# Space after open -- post-char rejects (matches bold/italic/strike).
- _ not under_
# Bare `_` followed by EOL -- post-char rejects.
- _
